# knowledge

这是小幽的知识层。

它和 `memory/` 的区别：
- `memory/` 记录每天发生了什么、做了什么、学到了什么，偏日志与短期上下文
- `knowledge/` 存放可以长期复用、整理、引用的知识，偏结构化沉淀

## 目录说明

- `reference/`：参考资料、工具说明、平台规则、稳定备忘
- `lessons/`：经验教训、踩坑复盘、最佳实践
- `safety/`：安全边界、隐私规则、群聊/私聊边界、风险案例
- `decisions/`：关键决策记录，说明“为什么这么定”
- `projects/`：项目文档与项目模板

## 使用原则

1. 新的长期知识，优先写到这里，而不是堆进 `MEMORY.md`
2. daily memory 中有价值的内容，后续提炼后迁移到这里
3. 这里是知识库，不是临时输出目录；临时文件请放 `tmp/`
4. 项目资料与助手本源分开管理，避免 workspace 根目录膨胀
